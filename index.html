Telnet: инструкция по применению

Пункт 0: добавление файлов на сервер

С помощью команды git clone загружаем файлы в папку backend2 на сервере. С помощью приложения FileZilla проверяем наличие файлов в нужной папке.

Пункт 1: соединение с сервером

Telnet - сетевой протокол для реализации текстового терминального интефейса по сети. Чтобы установить соединение с сервером с помощью протокола telnet, нужно ввести IP-адрес сервера, выбрать тип соединения - Raw, указать 80 порт и запретить закрытие окна после разрыва соединения.

Пункт 2: главная страница методом GET в протоколе HTTP 1.0

Чтобы загрузить страницу, используем метод GET. После / нужно указать адрес ресурса, так как нам нужна главная страница, после / поставим пробел. При использовании протокола HTTP 1.0 не нужно указывать доменное имя сервера.

Пункт 3: внутренняя страница методом GET в протоколе HTTP 1.1

Чтобы загрузить страницу, используем метод GET. После / введём адрес нужной нам внутренней страницы, например, /backend2. При использовании протокола HTTP 1.1 нужно указать доменное имя сервера, так как один сервер может поддерживать несколько хостов. Сервер отвечает статусом: 404 Not Found, так как по умолчанию загружается файл с именем index, в нашем случай был загружен файл index.php.

Пункт 4: размер файла file.tar.gz

Чтобы узнать информацию о файле, не загружая его, используем метод HEAD, позволяющий посмотреть заголовки без сущности. Заголовок Content-Lenght содержит информацию о размере файла в байтах. Файл file.tar.gz имеет размер 11335 байт.

Пункт 5: медиатип ресурса image.png

Для определения медиатипа также воспользуемся методом HEAD. Заголовок Content-Type содержит информацию о MIME типе данных. Простейший MIME тип состоит из типа и подтипа — двух строк разделённых /, без использования пробелов. Файл image.png - это изображение с расширением png.

Пункт 6: комментарий на сервер по адресу index.php

Чтобы отправить информацию на сервер, используем метод POST. В заголовке Content-Type укажем тип application/x-www-form-urlencoded, который используют простые формы. В заголовке Content-Lenght укажем размер сущности в байтах. В данном примере значение поля field1 должно быть равно value1, а field2 - value2.

Пункт 7: первые 100 байт файла file.tar.gz

Чтобы загрузить определённую часть файла, в запросе нужно добавить заголовок Range, в котором нужно указать нужный диапозон в байтах. Сервер отвечает статусом: 206 Partial Content. В Content-Length заголовке теперь указан размер запрошенного диапазона (а не полный размер файла). Заголовок Content-Range ответа указывает, где в полном ресурсе находится это частичное сообщение.

Пункт 8: кодировку ресурса index.php

Для определения медиатипа также воспользуемся методом HEAD. В заголовоке Content-Type после указания типа и подтипа данных можно увидеть параметр charset, который указывает кодировку ресурса. Если этого параметра нет, то по умолчанию используется кодировка ASCII. У файла index.php кодировка UTF-8.
